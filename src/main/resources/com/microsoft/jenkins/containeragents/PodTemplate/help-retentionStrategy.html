Container Once Retention Strategy
For each job in the queue, a new agent is started. Once the job has finished, the agent is shut down.

Container Idle Retention Strategy
Based on the workload provided by the queue (load average), new agents are started on demand. After the job(s) have finished, the agent is not shut down immediately.
If no new job was executed on this agent during the period of the Idle delay, the agent is shut down. Retention time 0 means never being deleted automatically.
